# 04 実装計画・検収・レビュー仕様書

## 1. ワークパッケージ（WP）と依存関係

```
WP0 ──> WP1 ──> WP2 ──┬──> WP4 ──> WP5(本文執筆) ──> WP6(統合)
              WP3 ────┘            ↑
              (検証CI)             WP4のfigures/values.jsonを参照
```

### WP0: リポジトリ雛形（0.5日相当）
- 構成: `cmbcore/ tests/ notebooks/ scripts/ textbook/(LaTeX) figures/ specs/(本仕様のコピー)`。
- `pyproject.toml`（依存固定）、`Makefile`: `make test / figures / notebooks / book / all`。
- LaTeX雛形: jsbook or lualatex+jlreq、200p想定のスタイル（本文10.5pt、definition/box環境3種、cleveref、図は `figures/` 参照）。
- 検収: `make all` がスタブ状態で完走。

### WP1: `cmbcore` 実装（`03_numerics_spec.md` 全節）
- 実装順: constants/params → background → recombination → perturbations(TCなし→TC追加) → spectrum → analytic。各段で §5.1–5.2 のテストを先に書く（TDD）。
- 検収: pytest 全緑、§5.2 特徴量表クリア、性能目標達成。

### WP2: ノートブック9冊（NB0–NB8）（`02_notebook_spec.md`）
- 検収: nbconvert完走、各NBの検収値クリア、図ファイル全数出力、スライド再現図の定性一致（レビューエージェントが添付スライドと目視比較し所見を残す）。

### WP3: 検証CI
- GitHub Actions相当: pytest＋NB実行（キャッシュ利用）＋CLASS比較（キャッシュCSVモード）。`values.json`（$z_*, r_s, \ell_1$, ピーク比, 応答率など本文が差し込む全数値）を生成する `scripts/make_values.py` を含む。
- 検収: CIが緑、`values.json` のスキーマ（キー一覧を本WPで確定しWP5に引き渡す）。

### WP4: 図版パイプライン
- `scripts/make_figures.py` が本文参照図（約60点; `01` の図リスト）を全生成。図番号→ファイル名対応表 `figures/manifest.yaml`。
- 検収: 全図存在、manifest と `01` の図リストの突合せ100%。

### WP5: 本文執筆（`01_textbook_spec.md`）
- 章ごとに1タスク。**執筆順は 1→2→3→6→5→4→7→8→9→10→11→12→14→13→付録**（6章を先に書いて記法のストレステストを行い、4–5章を遡って整合させる。13章は全数値が出揃う最後）。
- 各章の納品物: `textbook/chXX.tex` ＋「導出契約履行表」（契約ID／本文式番号／検算者欄）。
- 検収: 導出契約全履行、ページ予算±15%、`values.json` 差し込みのみで数値が書かれていること（grep監査: 本文中の裸の数値リテラルを検出するスクリプトをWP3で提供）。

### WP6: 統合・最終ビルド
- 相互参照解決、索引、記法表の自動生成（シンボルレジストリ `notation.yaml` から）、通しビルド、§5チェックリスト実施。
- 検収: `00_README.md` §5 の Definition of Done 全項目。

## 2. 各エージェントへの役割プロンプト要約

- **実装エージェント（WP1–4）**: 「`03`/`02` を唯一の仕様とする。仕様と典拠（Callin/Winther）の不一致を見つけたら実装せず Issue 化。物理の独自判断・式の創作を禁止。」
- **執筆エージェント（WP5）**: 「`01` の導出契約を満たす。式は `03` と文字単位一致。数値は `values.json` のみ。1ステップ規則（学部4年生5分検算）厳守。」
- **レビューエージェント（WP6・常駐）**: 「§5チェックリストを章単位で適用。不合格は契約ID単位で差し戻し。」

## 3. 検証プロトコル（数値）

1. **特徴量検証**: `03` §5.2 の表。実装完了後、CLASS同等設定の値で表を再較正し、PR上で新旧併記して確定。
2. **CLASS比較**: `03` §5.3 設定A。比較プロットと最大相対差を `validation_report.md` に自動出力。
3. **収束検証**: $\ell_{\max}\in\{6,8,12\}$, $n_k\in\{150,250,400\}$, rtol $\in\{10^{-6},10^{-8}\}$ の感度。$C_\ell$ 変化が許容（1%）に収まる最小設定を既定値として採用し、第14章に記載。
4. **スライド再現検証**: NB3/NB5 の8図を添付スライド p19–23, 25–26, 31, 33–34 と並置した比較シートを生成（`validation_report.md` 添付）。位相・振動回数・成長開始順序の一致を所見として記録。完全一致は要求しない（スライドはパラメータが旧WMAP値のため）。再現図はスライドのパラメータ（Ωb=0.046,Ωm=0.224,h=0.7,massless ν, n=1）プリセット `Params.callin2006()` で計算すること。

## 4. リスク登録簿と対応

| リスク | 兆候 | 対応 |
|---|---|---|
| 符号・係数の転記ミス（最頻出） | CLASS比で定数倍/位相ずれ | §5.1式テスト、典拠突合せ、`R` と `R_s` の混同チェック |
| 規格化ミス（$4\pi$, $(2\ell+1)$, $\mathcal{R}$換算） | $C_\ell$ が一様に数倍ずれる | 11.3 規格化監査＋SWプラトー解析値との比較 |
| TC切替の不連続 | $\Theta_0$ にキンク、減衰尾の偽振動 | 切替条件の感度試験、連続性assert |
| ISWの積分不足 | 低 $\ell$ がCLASSと>5%ずれ | 後期 $x$ 刻み・$k_{\min}$ の収束テスト |
| ベッセル補間誤差 | 高 $\ell$ ピーク位置ずれ | $\Delta z$ 半減で不変性確認 |
| 偏光無視の系統誤差 | 減衰尾で数%系統差 | 仕様内（設定B比較で可視化し本文に明記） |

## 7. 本仕様書セルフレビュー記録（初版作成時に実施）

レビュー観点と結果:

1. **要求カバレッジ**: ユーザー要求7項目（FLRW起点／熱史／コンプトン散乱→CMB／線形摂動＋ボルツマン／音響振動／その他全要素の計算的説明／mν・τ・z*・z_re の定量変化、ノートブック実習）を §5-D に逆引きで配置し、全てに章・NB・図IDの割当があることを確認。**合格**。
2. **「最小構成・飛躍なし」原則の具体化**: 「導出契約＋1ステップ規則（5分検算）」として検収可能な形に翻訳。偏光・テンソル・非線形の除外と、その誤差の定量化義務（設定B比較、付録E）をセットにした。**合格**。
3. **記法の罠の事前潰し**: (a) Ma&Bertschinger と Dodelson/Callin の $\Phi,\Psi$ 符号差 → 付録A対応表を義務化。(b) $R_s$ と $R=1/R_s$ の混同（実際に頻発するバグ）→ `03` §2.3 と `04` §4 に明示。(c) 規格化（$\mathcal{R}$換算・$4\pi$）→ 第11.3節「規格化監査」を本文の節として設置。**初稿でR混同の説明が8章と3章で別表現だったため統一済み**。
4. **式の信頼性**: 本仕様内の方程式（P/N/M/G系、初期条件、TC式、ソース関数、Peebles係数）はCallin/Winther系の標準形で記載したが、転記ミスのリスクは消せないため「実装前の典拠突合せ＋式レベル単体テスト＋不一致時停止」を `03` 冒頭と本ファイル§6に義務として明文化。仕様自体を無謬と仮定しない設計とした。**合格（設計で担保）**。
5. **ページ予算**: 章別合計200p（190–215許容）。13章18p・6章20pが最大であり、肥大リスクは §4 のリスク登録簿で吸収。**合格**。
6. **図とスライドの対応**: 添付スライドの教育図8点（p19,20,21,22,23,25,26,31,33,34）全てにNB図IDを割当（F3.1–F3.6, F5.1, F5.5）。スライドが旧WMAP系パラメータである点は `Params.callin2006()` プリセットで解決。**初稿でこのプリセット指定が漏れていたため §3-4 に追記済み**。
7. **下流AIの実行可能性**: 各WPに「入力仕様ファイル・納品物・検収条件」が揃っているか走査。WP3の `values.json` スキーマ確定をWP5の前提として依存グラフに明示。**合格**。
8. **検証値の現実性**: 特徴量表（$z_*=1090$, $r_s=144.5$ Mpc, $100\theta_*=1.041$, $\ell_1=220$, $\ell_2/\ell_1\approx2.45$, $k_D\approx0.14$ Mpc⁻¹）は Planck 2018 近傍の標準値。ただし $Y_p=0$ 等の教育的単純化により数%ずれうるため、「実装後にCLASS同等設定で再較正して確定」とする逃げ道を §3-1 に設けた。**合格**。
